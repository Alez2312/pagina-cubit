¿Qué permite la implementacion de un interceptor?

La implementacion de un interceptor nos permite encapsular código de una forma sencilla y aplicarlo a nuestros métodos sin interferir directamente, también proporcioana un mecanismo simple y configurable para aplicar la conducta en el lugar que deseemos.

¿Cuándo se implementa un guard?

Los guards se implementan para ser inyectandos por lo tanto tenemos que usar la etiqueta @Inyectable, como si fuera un servicio.

¿Qué es una prueba unitaria?

Una prueba unitaria es ina forma de probar el funcionamiento de un código.

¿Qué es una prueba de servicio?

Las pruebas de servicios son las que dan soporte al mecanismo de autenticación simple y doble.

¿Qué es una prueba de componente?

Las pruebas de componentes se ejecutan de forma independiente para comprobar que el resultado sea el requerido.

Uso de useEffect

Se usa para añadir un efecto que se ejecutará cada vez que nuestro componente se renderize

Uso de useState

Se usa para añadir un estado interno a nuestros componentes para hacerlos dinámicos e interactivos.
